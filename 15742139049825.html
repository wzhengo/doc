泛型

2019/11/20 posted in  Java知识点归类

什么是泛型？

泛型是一种语法糖。Java中最常用的语法糖主要有泛型、变长参数、条件编译、自动拆装箱、内部类等。

泛型有什么作用？

  • 告诉编译器帮我们检查类型是否匹配；
  • 使用时自动类型转换，避免ClassCastException。

泛型的使用

  • 泛型类和泛型接口
  • 泛型方法
  • 泛型构造器

泛型派生子类时不能再包含类型形参，需要传入具体的类型。

//定义接口时指定了一个类型形参，该形参名为E
public interface List<E> extends Collection<E> {
   //在该接口里，E可以作为类型使用
   public E get(int index) {}
   public void add(E e) {} 
}

//定义类时指定了一个类型形参，该形参名为E
public class ArrayList<E> extends AbstractList<E> implements List<E> {
   //在该类里，E可以作为类型使用
   public void set(E e) {
   ...
   }
}

//泛型构造器
public class Person {
    public <T> Person(T t) {
    System.out.println(t);
    }
}

什么是类型擦除

  • 编译期进行类型擦除，变为原始类型。
  • 虚拟机没有泛型类型对象，所有对象都是普通类。

通配符上下边界

  • <? extends T> 上界通配符 所有的类都继承T
  • <? super T> 下界通配符 所有的类都是T的父类
  • <?> 无界通配符 任意类型，不能是Object，只能是null

泛型的限制

  • 泛型不能是基本类型
  • 运行时类型查询instanceof只能查到原始类型
  • 不能创建参数化类型的数组
  • 不能构造泛型数组
  • 泛型类的静态变量无效
  • 不能创建泛型异常类

RxJava中的泛型

慕课-Java泛型的实现机制是怎样的？

  • 使用类型擦除实现，类型编译时被擦除为Object
  • 类型擦除有哪些好处？
    • 运行时内存负担小，虚拟机中只有一种类型
    • 兼容性好，Java1.5才推出
  • 类型擦除有哪些限制？
    • 基本类型无法作为泛型实参
    • 泛型类型无法用作方法重载，因为类型被擦除了
    • 泛型类型无法当做真实类型使用
      • Gson.fromJson为什么需要传入Class？因为类型被擦除了
    • 静态方法无法引用类泛型参数
    • 类型强转的运行时开销
  • 附加的签名信息
    • 混淆时要保留签名信息
    • 使用泛型签名的两个实例
      • Gson的TypeToken<>
      • Retrofit进行网络请求的Call<>
    • Kotlin反射的实现原理，用注解实现和泛型
