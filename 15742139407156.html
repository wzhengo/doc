注解

2019/11/20 posted in  Java知识点归类

什么是注解

  • 注解是代码里的特殊标记，这些标记可以在编译、类加载、运行时被读取，并执行相应的处理。
  • 通过使用注解，开发人员可以在不改变原有逻辑的情况下，在源文件中嵌入一些补充的信息。
  • 注解只是一种 metadata(元数据)传递的渠道，本身并没有实现功能。

注解分类

1.标准注解

@Override、@Deprecated（反对）、@SuppressWarnings（抑制警告）、@SafeVarargs（可变长度参数）

2.元注解

  • @Targe（目标，可用范围）
  • @Inherited（被继承）
  • @Documented（文档）
  • @Retention（保留策略）
    • RetentionPolicy.SOURCE：Java文件
    • RetentionPolicy.CLASS：Class文件
    • RetentionPolicy.RUNTIME：JVM中
  • @Repeatable（允许多次使用）

Android Annotation

  • Nullness注解
    • @NonNull 不可空
    • @Nullable 可为空
  • Resource Type注解
    • @StringRes
    • @IntRes
  • Threading 注解
    • @UiThread
    • @WorkThread
  • Overriding Methods注解
    • @CallSuper

自定义注解

1.基本定义

用@interface定义注解：

public @interface SpiderMan{
}

使用注解：

public class AnnotationTest{
    @SpiderMan
    public void test(){
        ...
    }
}

2.定义成员变量

定义成员变量：

public @interface SpiderMan{
    String name();
    int age() default 18;
}

使用：

public class AnnotationTest{
    @SpiderMan(name = "abc")
    public void test(){
        ...
    }
}

3.定义运行时注解

@Retention(RetentionPolicy.RUNTIME)
public @interface SpiderMan{
    ...
}

4.定义编译时注解

@Retention(RetentionPolicy.class)
public @interface SpiderMan{
    ...
}

1.运行时注解处理器
通过反射获得Field、Method等信息，调用getAnnotation()方法即可获得注解信息。

注解的使用

1.反射，运行时动态获取注解信息
2.APT，编译时注解处理器
