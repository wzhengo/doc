什么是反射？反射的原理

  • 反射是为了能够动态的加载一个类、调用方法、访问属性等动态要求而设计的。
  • JVM会为每个类创建一个java.lang.Class类的实例，通过该对象可以获取这个类的信息，然后通过使用java.lang.reflect包下得API以达到各种动态需求。
  • 反射机制是在运行状态中：
    • 对于任意一个类，都能够知道这个类的所有属性和方法；
    • 对于任意一个对象，都能够调用它的任意一个方法和属性。

如何使用反射

获得类对象

Class person =

  • Class.forName("com.aaa.Person");
  • Person.class;
  • person.getClass();

获得变量、方法等

  • java.lang.Class.java
    • getField、getMethod...
  • java.lang.reflect.Field.java
    • get()、set()
  • java.lang.reflect.Method.java
    • invoke()
  • java.lang.reflect.Constructor.java
    • newInstance()`

注意：

  • 私有变量或方法
    • getDeclaredField、getDeclaredMethod
    • setAccessible(true)
  • 静态变量或方法使用get(null)、invoke(null,...)

反射的应用

反射的用途

官方解释：反射被广泛地用于那些需要在运行时检测或修改程序行为的程序中。

应用场景

  • 需要访问隐藏属性或者调用方法改变程序原来的逻辑，使用没有系统并没有开放一些接口出来
  • 自定义注解，注解就是在运行时利用反射机制来获取的
  • 在开发中动态加载类，比如在Android中的动态加载解决65k问题等等，模块化和插件化等

扩展：编译时vs运行时

  1. 编译时vs运行时

    • 编译时：将Java代码编译成.class文件的过程
    • 运行时：Java虚拟机执行.class文件的过程

  2. 编译时类型vs运行时类型

    • 编译时类型：编译时类型由声明该变量时的类型决定
    • 运行时类型：运行时类型由实际赋给该变量的类型决定
    //Animal是编译时类型，Dog是运行时类型
Animal animal = new Dog();

  3. 动态绑定-调用引用实例的方法

    1. 编译时，是调用声明类型的成员方法，即编译时类型的方法
    2. 运行时，调用的是实际的类型的成员方法，即运行时类型的方法。
    3. 对于调用引用实例的成员变量，无论是编译时还是运行时，均是调用编译时类型的成员变量

面试题

反射的原理是什么？有哪些途径获取到Class对象，Class类的含义和作用是什么？什么是class类？

  • Class文件：是由一个.java文件通过编译过程生成的.class文件。
  • Class对象：当Java虚拟机加载一个.class文件到内存时，都会生成一个对应的Class对象，注意的是这个Class对象有且只有一个存在于内存当中。

有哪些方式可以提高反射效率？为何反射消耗性能？究竟是怎么影响的，举例说明？

提高反射效率：

  • 适当使用缓存，比如，需要多次动态创建一个类的实例的时候，有缓存的写法会比没有缓存要快很多。

为何反射消耗性能？

  1. 由于是本地方法调用，让JVM无法优化(还有JIT)。
  2. 反射方法调用还有验证过程和参数问题，参数需要装箱拆箱、需要组装成Object[]形式、异常的包装等等问题。

如何优化？

  1. JIT
  2. ReflectASM反射库

java反射机制提供了什么功能？反射具有暴力访问权限，如何防止反射序列化攻击单例？

防止反射序列化攻击单例：

  • 枚举
  • abstract

通过反射获得泛型的实际类型参数？反射获取构造方法，变量，方法的方法是哪些？

  • Method类的getGenericParameterTypes
  • ParameterizedType类的getRawType、getActualTypeArguments。

getGenericParameterTypes 与 getParameterTypes区别？

都是java.lang.reflect.Executable的方法。

//Executable.java
public Type[] getGenericParameterTypes() {
    if (hasGenericInformation())
        return getGenericInfo().getParameterTypes();
    else
        return getParameterTypes();
}
