什么是反射？反射的原理

  • 反射是为了能够动态的加载一个类、调用方法、访问属性等动态要求而设计的。
  • JVM会为每个类创建一个java.lang.Class类的实例，通过该对象可以获取这个类的信息，然后通过使用java.lang.reflect包下得API以达到各种动态需求。
  • 反射机制是在运行状态中：
    • 对于任意一个类，都能够知道这个类的所有属性和方法；
    • 对于任意一个对象，都能够调用它的任意一个方法和属性。

如何使用反射

获得类对象

Class person =

  • Class.forName("com.aaa.Person");
  • Person.class;
  • person.getClass();

获得变量、方法等

  • java.lang.Class.java
    • getField、getMethod...
  • java.lang.reflect.Field.java
    • get()、set()
  • java.lang.reflect.Method.java
    • invoke()
  • java.lang.reflect.Constructor.java
    • newInstance()`

注意：

  • 私有变量或方法
    • getDeclaredField、getDeclaredMethod
    • setAccessible(true)
  • 静态变量或方法使用get(null)、invoke(null,...)

反射的应用

反射的用途

官方解释：反射被广泛地用于那些需要在运行时检测或修改程序行为的程序中。

应用场景

  • 需要访问隐藏属性或者调用方法改变程序原来的逻辑，使用没有系统并没有开放一些接口出来
  • 自定义注解，注解就是在运行时利用反射机制来获取的
  • 在开发中动态加载类，比如在Android中的动态加载解决65k问题等等，模块化和插件化等

扩展：编译时vs运行时

  1. 编译时vs运行时

    • 编译时：将Java代码编译成.class文件的过程
    • 运行时：Java虚拟机执行.class文件的过程

  2. 编译时类型vs运行时类型

    • 编译时类型：编译时类型由声明该变量时的类型决定
    • 运行时类型：运行时类型由实际赋给该变量的类型决定
    //Animal是编译时类型，Dog是运行时类型
Animal animal = new Dog();

  3. 动态绑定-调用引用实例的方法

    1. 编译时，是调用声明类型的成员方法，即编译时类型的方法
    2. 运行时，调用的是实际的类型的成员方法，即运行时类型的方法。
    3. 对于调用引用实例的成员变量，无论是编译时还是运行时，均是调用编译时类型的成员变量

泛型

什么是泛型？

泛型是一种语法糖。Java中最常用的语法糖主要有泛型、变长参数、条件编译、自动拆装箱、内部类等。

泛型有什么作用？

  • 告诉编译器帮我们检查类型是否匹配；
  • 使用时自动类型转换，避免ClassCastException。

泛型的使用

  • 泛型类和泛型接口
  • 泛型方法
  • 泛型构造器

泛型派生子类时不能再包含类型形参，需要传入具体的类型。

//定义接口时指定了一个类型形参，该形参名为E
public interface List<E> extends Collection<E> {
   //在该接口里，E可以作为类型使用
   public E get(int index) {}
   public void add(E e) {} 
}

//定义类时指定了一个类型形参，该形参名为E
public class ArrayList<E> extends AbstractList<E> implements List<E> {
   //在该类里，E可以作为类型使用
   public void set(E e) {
   ...
   }
}

//泛型构造器
public class Person {
    public <T> Person(T t) {
    System.out.println(t);
    }
}

什么是类型擦除

  • 编译期进行类型擦除，变为原始类型。
  • 虚拟机没有泛型类型对象，所有对象都是普通类。

通配符上下边界

  • <? extends T> 上界通配符 所有的类都继承T
  • <? super T> 下界通配符 所有的类都是T的父类
  • <?> 无界通配符 任意类型，不能是Object，只能是null

泛型的限制

  • 泛型不能是基本类型
  • 运行时类型查询instanceof只能查到原始类型
  • 不能创建参数化类型的数组
  • 不能构造泛型数组
  • 泛型类的静态变量无效
  • 不能创建泛型异常类

注解

什么是注解

  • 注解是代码里的特殊标记，这些标记可以在编译、类加载、运行时被读取，并执行相应的处理。
  • 通过使用注解，开发人员可以在不改变原有逻辑的情况下，在源文件中嵌入一些补充的信息。
  • 注解只是一种 metadata(元数据)传递的渠道，本身并没有实现功能。

注解分类

1.标准注解

@Override、@Deprecated（反对）、@SuppressWarnings（抑制警告）、@SafeVarargs（可变长度参数）

2.元注解

  • @Targe（目标，可用范围）
  • @Inherited（被继承）
  • @Documented（文档）
  • @Retention（保留策略）
    • RetentionPolicy.SOURCE：Java文件
    • RetentionPolicy.CLASS：Class文件
    • RetentionPolicy.RUNTIME：JVM中
  • @Repeatable（允许多次使用）

Android Annotation

  • Nullness注解
    • @NonNull 不可空
    • @Nullable 可为空
  • Resource Type注解
    • @StringRes
    • @IntRes
  • Threading 注解
    • @UiThread
    • @WorkThread
  • Overriding Methods注解
    • @CallSuper

自定义注解

1.基本定义

用@interface定义注解：

}

使用注解：

定义成员变量：

I/O

Java 中有三代 IO 框架：

  1. 第一代的流式阻塞 IO (Blocking IO)；
  2. 第二代的 NIO (New IO)是非阻塞的，java 1.4引入。提供了 Channel、Selector、Buffer 等新的抽象，可以构建多路复用的、同步非阻塞 IO 程序，同时提供了更接近操作系统底层的高性能数据操作方式。
  3. 第三代 NIO2(有些地方叫做AIO，即 async IO) 又进一步支持了异步 IO，java 1.7引入。异步 IO 操作基于事件和回调机制，应用操作直接返回，而不会阻塞在那里，当后台处理完成，操作系统会通知相应线程进行后续工作。

字节与字符

字节与字符

  • 每个字节(byte)有8bit组成，每种数据类型又几个字节组成等。
  • 字符：代表一个汉字或者英文字母。字符流 = 字节流 + 编码表。

字节流和字符流的区别：

  • 读写单位不同：字节流以字节（8bit）为单位，字符流以字符为单位，根据码表映射字符，一次可能读多个字节。
  • 处理对象不同：字节流能处理所有类型的数据（如图片、avi等），而字符流只能处理字符类型的数据，比如文本内容。

常量流分类

  1. InputStream字节输入流
  2. OutputStream字节输出流
  3. Reader字符输入流
  4. Writer字符输出流

NIO

标准的IO基于字节流和字符流进行操作的，而NIO是基于通道(Channel)和缓冲区(Buffer)进行操作，数据总是从通道读取到缓冲区中，或者从缓冲区写入通道也类似。

Buffer

利用Buffer读写数据，通常遵循四个步骤：

  1. 把数据写入buffer；
  2. 调用flip；
  3. 从Buffer中读取数据；

  4. 调用buffer.clear()

  • Buffer有三个重要属性：容量，位置，上限（Buffer Capacity, Position and Limit）
  • 分配Buffer：allocate()

Channel

Java NIO Channel通道和流非常相似，主要有以下几点区别：

  • 通道是双向的，可以读也可以写；流一般来说是单向的，只能读或者写。
  • 通道可以异步读写。
  • 通道总是基于缓冲区Buffer来读写。

Channel的实现类有：

  • FileChannel
  • DatagramChannel
  • SocketChannel
  • ServerSocketChannel

区分同步/异步 和 阻塞/非阻塞

  • 同步与异步（synchronous/asynchronous）
    • 同步是一种可靠的有序运行机制，当我们进行同步操作时，后续的任务是等待当前调用返回，才会进行下一步；
    • 而异步则相反，其他任务不需要等待当前调用返回，通常依靠事件、回调等机制来实现任务间次序关系。
  • 阻塞与非阻塞（blocking/non-blocking）
    • 在进行阻塞操作时，当前线程会处于阻塞状态，无法从事其他任务，只有当条件就绪才能继续，比如 ServerSocket 新连接建立完毕，或数据读取、写入操作完成；
    • 而非阻塞则是不管 IO 操作是否结束，直接返回，相应操作在后台继续处理。

Selector

  1. 首先，通过 Selector.open() 创建一个 Selector，作为类似调度员的角色。
  2. 然后，创建一个 ServerSocketChannel，并且向 Selector 注册，通过指定 SelectionKey.OP_ACCEPT，告诉调度员，它关注的是新的连接请求。注意，为什么我们要明确配置非阻塞模式呢？这是因为阻塞模式下，注册操作是不允许的，会抛出 IllegalBlockingModeException 异常。
  3. Selector 阻塞在 select 操作，当有 Channel 发生接入请求，就会被唤醒。
  4. 在 sayHelloWorld 方法中，通过 SocketChannel 和 Buffer 进行数据操作，在本例中是发送了一段字符串。

public class NIOServer extends Thread {
    public void run() {
        try (Selector selector = Selector.open();
             ServerSocketChannel serverSocket = ServerSocketChannel.open();) {// 创建Selector和Channel
            serverSocket.bind(new InetSocketAddress(InetAddress.getLocalHost(), 8888));
            serverSocket.configureBlocking(false);
            // 注册到Selector，并说明关注点
            serverSocket.register(selector, SelectionKey.OP_ACCEPT);
            while (true) {
                selector.select();// 阻塞等待就绪的Channel，这是关键点之一
                Set<SelectionKey> selectedKeys = selector.selectedKeys();
                Iterator<SelectionKey> iter = selectedKeys.iterator();
                while (iter.hasNext()) {
                    SelectionKey key = iter.next();
                   // 生产系统中一般会额外进行就绪状态检查
                    sayHelloWorld((ServerSocketChannel) key.channel());
                    iter.remove();
                }
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
    private void sayHelloWorld(ServerSocketChannel server) throws IOException {
        try (SocketChannel client = server.accept();) {          client.write(Charset.defaultCharset().encode("Hello world!"));
        }
    }

}

深拷贝和浅拷贝

浅拷贝

  • 浅拷贝是按位拷贝对象，它会创建一个新对象，这个对象有着原始对象属性值的一份精确拷贝。
  • 如果属性是基本类型，拷贝的就是基本类型的值；
  • 如果属性是内存地址（引用类型），拷贝的就是内存地址。

实现浅拷贝：
实现了Clonable接口并重写Object类的clone()方法，然后在方法内部调用super.clone()方法。

public Object clone() {
    //浅拷贝
    try {
        // 直接调用父类的clone()方法
        return super.clone();
    } catch (CloneNotSupportedException e) {
        return null;
    }
}

深拷贝

深拷贝会拷贝所有的属性,并拷贝属性指向的动态分配的内存。当对象和它所引用的对象一起拷贝时即发生深拷贝。深拷贝相比于浅拷贝速度较慢并且花销较大。

实现深拷贝：
实现了Clonable接口并重写Object类的clone()方法，然后创建拷贝类的一个新对象。

public Object clone() { 
   // 深拷贝，创建拷贝类的一个新对象，这样就和原始对象相互独立
   Student s = new Student(name, subj.getName()); 
   return s; 
}
